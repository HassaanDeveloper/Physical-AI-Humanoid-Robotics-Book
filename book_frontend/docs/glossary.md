# Glossary of Terms

## A

**Actuator**: A mechanical device for moving or controlling a mechanism

**Architecture**: The fundamental structure of a digital twin system

## B

**Bidirectional**: Moving in two directions; refers to data flow between physical and virtual systems

## C

**Collision Detection**: The computational problem of detecting when solid objects touch each other

**Component**: A modular, portable, replaceable, and reusable set of well-defined interfaces

## D

**Digital Twin**: A virtual representation of a physical robot that mirrors its state and behavior in real-time

## E

**Environment**: The physical or virtual space in which a robot operates

## F

**Friction**: The resistance to motion of one object moving relative to another

## G

**Gazebo**: A 3D simulation environment for robotics applications

**Gravity**: The force that attracts a body toward the center of the earth, or toward any other physical body having mass

## H

**Humanoid**: Having human-like characteristics, especially in appearance or behavior

## I

**IMU**: Inertial Measurement Unit - measures and reports a body's specific force, angular rate, and sometimes magnetic fields

**Interface**: A shared boundary across which two or more separate components exchange information

## L

**LiDAR**: Light Detection and Ranging - a remote sensing method that uses light in the form of a pulsed laser

## P

**Physics Engine**: Software that provides an approximate simulation of physical phenomena

**Plugin**: A software component that adds a specific feature to an existing computer program

## Q

**Quality Assurance**: A way of preventing mistakes or defects in manufactured products and avoiding problems

## R

**Real-time**: Processing data as it becomes available, without delay

**ROS**: Robot Operating System - flexible framework for writing robot software

**ROS#**: A pure C# library that implements the ROS wire protocol

## S

**Sensor Fusion**: The combining of sensory data or data derived from disparate sources

**Simulation**: The imitation of the operation of a real-world process or system

**URDF**: Unified Robot Description Format - an XML format for representing a robot model

## U

**Unity**: A cross-platform game engine used for creating interactive 3D environments

## V

**Virtual**: Made, done, seen, etc., using computer software and not involving physical reality

**Visualization**: The act of interpreting in visual terms or the process of making something visible

## W

**World File**: A Gazebo file that describes the environment and objects in a simulation
